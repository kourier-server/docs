Hello Cute

  • Setup development environment
  • Create client UI app
  • Make client app use remote objects to fetch from the server the text to be shown in its UI.
  • Build client app for the desktop, iOS, Android and WebAssembly
  • Create the remote objects library.
  • Implement a remote object class that informs the text to be shown on the screen.
  • Configure the server to load the created library.
  • Run the server and the app and use the app to fetch from the server the text to be shown on its UI.

Our first tutorial does not do any network communication. In the Hello Qt tutorial, we create a Qt Quick-based app that prints "Hello Cute!" on the screen. It serves the purpose of showing how to implement Qt6 apps using CMake and QtCreator and, from a single source code, build desktop, iOS, Android and WebAssembly applications.

With Qt, developers use QtQuick and the QML language to create user interfaces declaratively. In this declarative UI paradigm, Qt applies signals and slots to model state changes as events that get propagated in the UI and can be used to trigger business logic or change other UI elements. The declarative paradigm for creating user interfaces took the world by storm, and following Qt all major companies embraced it. For example, React Native (Meta), Flutter (Google), SwiftUI (Apple), Jetpack Compose (Android) and Xamarin.Forms (Microsoft) are declarative UI frameworks that adopted the declarative style well after Qt. Qt provides a solid, modern and mature framework for developing fluid and responsive user interfaces.

With Qt, you apply signals and slots to communicate within your app, and, with Cute, you can use this powerful and intuitive communication mechanism to communicate over a network with remote peers.

The simple calculator app shows how to create iOS, Android, WebAssembly and Desktop clients that communicate to server to add two integers.

Simple Calculator

  1. Create a console app that receives from the command line the server IP/port and the two integers to add.
  2. Use the RemoteObject class to perform addition and print the result on the console before quiting the app.
  3. Add the remote SimpleCalculator class.
  4. Build the remote objects library.
  5. Create the server configuration file.
  6. Deploy the server to an AWS instance.
  7. Test the app.

Streaming File To Server

  1. Create a console app that accepts the file to be streamed to server as a command-line argument and the server IP and port
  2. Use the RemoteObject class and its readyToWrite signal to send data to the server.
  3. Create the FileStreamReceiver remote class.
  4. Build the remote objects library.
  5. Create the server configuration file.
  6. Deploy the server to an AWS instance.
  7. Stream files to server and measure the send rate and the memory comsumed by the client.
  8. Change the FileStreamReceiver class to add a delay.
  9. Rebuild the library and make the server use the new library.
  10. Deploy the same file again and verify the send rate and the memory consumption on the client side.

Streaming File To Clients

  1. Create a console app.
  2. Create the FileReceiver class and make it interact with the server to receive a file from it.
  3. Make the FileReceiver class accept a delay time.
  4. Create three instances of the FileReciever class. One with no delay, the other with a small delay and one with a greater delay.
  5. Create the FileStreamSender remote class.
  6. Use the readyToWrite signal to send data to clients only when no more data has to be written to the network layer.
  7. Build the remote objects library.
  8. Create the server configuration file.
  9. Deploy the server to an AWS instance.
  10. Stream files to clients.
  11. Check memory consumtion and send rate on the client-side.
