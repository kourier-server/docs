Writing Server Code

All classes inheriting from Cute::Server::RemoteObject can contain remote signals and slots. Remote signals and slots are standard Qt signals/slots that developers register on source files. For example, consider the ContactService class shown below:

//
// ContactService.h
//
#include <CuteServerSdk.h>
using Cute::Server::RemoteObject;
// ContactService is a class that will have signals and slots
// made available for remote signal/slot interaction, and thus
// must inherit from Cute::Server::RemoteObject.
class ContactService : public RemoteObject
{
Q_OBJECT
public slots:
void addContact(QString name, QString address);
QList<std::pair<QString,QString> contacts();
signals:
void newContact(const QString &name, const QString &address);
};
//
// ContactService.cpp
//
#include "ContactService.h"
// Remote signals and slots are registered in source files
Cute::registerRemoteSignal(&ContactService::newContact);
Cute::registerRemoteSlot(&ContactService::addContact);
Cute::registerRemoteSlot(&ContactService::contacts);

All signals and slots from ContactService were made available for remote signal-slot interaction. Like Qt, Cute supports custom types. To use custom types in remote signals and slots, developers have to make the type known to QMetaType by declaring it with Q_DECLARE_METATYPE (Cute supports all template-based Qt containers and most of the template-based containers from the C++ Standard Library, which Cute registers automatically). Then, users have to register the type with Cute. Custom types must be registered on both client and server sides. Let's modify the ContactService class to make it use the Contact custom type on remote signals and slots:

//
// ContactService.h
//
#include <CuteServerSdk.h>
using Cute::DataStream;
using Cute::Server::RemoteObject;
// Contact is a custom type used on
// remote signals and slots.
struct Contact
{
QString name;
QString address;
};
// All custom types must be registered
// with Qt meta-type system.
Q_DECLARE_METATYPE(Contact);
// Custom types must define stream operators for
// Cute::DataStream.
inline DataStream &operator<<(DataStream &stream, const Contact &contact)
{stream << contact.name; stream << contact.address; return stream;}
inline DataStream &operator>>(DataStream &stream, Contact &contact)
{stream >> contact.name; stream >> contact.address; return stream;}
// ContactService is a class that will have signals and slots
// made available for remote signal/slot interaction, and thus
// must inherit from Cute::Server::RemoteObject.
class ContactService : public RemoteObject
{
Q_OBJECT
public slots:
void addContact(QString name, QString address);
void addContact(const Contact &contact);
QList<Contact> contacts();
signals:
void newContact(const Contact &contact);
};
//
// ContactService.cpp
//
#include "ContactService.h"
// Both Custom types and remote signals and slots
// are registered in source files
Cute::registerCustomType<Contact>();
Cute::registerRemoteSignal(&ContactService::newContact);
Cute::registerRemoteSlot(qOverload<QString,QString>(&ContactService::addContact));
Cute::registerRemoteSlot(qOverload<const Contact&>(&ContactService::addContact));
Cute::registerRemoteSlot(&ContactService::contacts);
